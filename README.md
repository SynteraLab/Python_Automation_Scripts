# Python Projects Collection

This repository contains a collection of personal Python-based projects, utilities, experiments, and supporting web/frontend work.  
The projects cover automation, Telegram tools, subtitle generation, media processing, scraping, monitoring, and web interface experiments.

## Overview

The repository is organized as a multi-project workspace.  
Each folder represents a separate application, prototype, or utility with its own purpose, dependencies, and runtime files.

Some projects are production-oriented tools, while others are learning experiments, drafts, or upgraded versions of older code.

## Repository Structure

This repository may contain the following file categories:

- source code files such as `.py`, `.js`, `.ts`, `.tsx`, and `.html`
- configuration files such as `.json`, `.yml`, `.yaml`, `.toml`, and `.env`
- runtime data such as logs, generated metadata, and cached state
- environment folders such as `.venv`, `venv`, or `env`
- frontend dependencies such as `node_modules`
- temporary or generated files such as `__pycache__`, `.DS_Store`, and build artifacts

## Notes

- Some folders include runtime-generated files such as logs, sessions, cache, and local environment data.
- Some projects may require separate setup because they were developed independently.
- Dependency folders and local virtual environments are included only for development history and may not be required for clean deployment.
- This repository is a mixed workspace rather than a single packaged application.

## Suggested Cleanup For Public Repositories

Before publishing or sharing publicly, it is recommended to remove or ignore:

- environment files
- session files
- local logs
- cache files
- virtual environments
- build output
- dependency folders
- machine-specific files such as `.DS_Store`
